## 🔄 二、运行逻辑

### 1. 消息流程

```
用户发送消息
    ↓
Gateway 接收 (WhatsApp/Telegram/...)
    ↓
验证 + 路由到对应 Session
    ↓
构建 Context (记忆 + 技能 + 历史)
    ↓
调用 LLM (MiniMax/GPT/Claude...)
    ↓
LLM 推理 + 决定工具调用
    ↓
执行工具 (exec/search/...)
    ↓
流式输出回复
    ↓
持久化到 Memory
```

### 2. Agent Loop (智能体循环)

```
┌─────────────────────────────────────────────┐
│ 1. Intake (接收)                           │
│    - 解析用户消息                            │

└─────────────────────────────────────────────┘
                     ↓
┌─────────────────────────────────────────────┐
│ 2. Context Assembly (上下文组装)            │
│    - System Prompt                          │
│    - Skills                                 │
│    - Memory                                 │
│    - Session History                        │
└─────────────────────────────────────────────┘
                     ↓
┌─────────────────────────────────────────────┐
│ 3. Model Inference (模型推理)               │
│    - 选择模型                                │
│    - 生成回复/决定工具                       │
└─────────────────────────────────────────────┘
                     ↓
┌─────────────────────────────────────────────┐
│ 4. Tool Execution (工具执行)                 │
│    - exec (命令执行)                        │
│    - read/write (文件操作)                  │

└─────────────────────────────────────────────┘
                     ↓
┌─────────────────────────────────────────────┐
│ 5. Streaming Reply (流式回复)               │

└─────────────────────────────────────────────┘
                     ↓
┌─────────────────────────────────────────────┐
│ 6. Persistence (持久化)                     │
│    - 保存到 SQLite                          │
```
